# SOUL.md

this file defines the project's personality, voice, and design philosophy.
it's not just for AI agents — it's for anyone who works on this project.

if this project was a person, what would it feel like?

## the vibe

how should this project feel to users?

- friendly but not childish?
- professional but not corporate?
- playful but not distracting?

be specific. "good UX" is meaningless. "warm, minimal, no jargon" is useful.

## design principles

3-5 principles that guide every decision.

examples:
- clarity over cleverness
- fast feedback over perfect design
- accessible to beginners, not insulting to experts
- every error message tells the user what to do next

## tone

what language should the UI use?

- "you're all set" vs "operation completed successfully"
- "something went wrong — here's how to fix it" vs "error 500"
- lowercase-friendly? emoji-allowed?

## anti-patterns

what should this project NEVER do?

- no loading spinners that last more than 2 seconds
- no error messages without action steps
- no jargon without explanation
